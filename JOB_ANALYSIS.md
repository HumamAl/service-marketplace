## Analysis Notes

### Domain
`marketplace` — two-sided service marketplace with strong SaaS/tech overlay (AI-first tooling, real-time systems, subscription billing). The "Airbnb meets Calendly" framing is accurate: discovery + booking + scheduling + payments in one platform.

### Why Variant C ("Similar Build")
This client explicitly requests a link to a marketplace/booking platform as the first screening question. Variant C leads with the closest existing portfolio match (Lynt Marketplace + Sienna Charles) and the working demo as proof. The cover letter opening should reference a specific prior build and link the demo in the same breath. The `REDESIGN` signal confirms this is extending existing work — not greenfield — which Variant C handles well ("I've done this before, here's what that looked like").

### Portfolio Project Ranking Logic
1. **Lynt Marketplace (#15)** — Direct domain match. Two-sided marketplace, vendor onboarding, listing management, transaction tracking. Best primary pick.
2. **Sienna Charles — Vendor Admin (#8)** — Highest feature overlap: AI-powered search, booking management, map-based discovery, spend analytics. Vendor = provider in this context.
3. **Rental PM Connect (#14)** — Two-sided platform with separate role-based dashboards (owner vs PM = client vs provider). Matching algorithm, vetting workflow, review system all directly applicable.
4. **Creator Economy App (#22)** — Stripe Connect payment architecture is the closest match to the subscription + deposit payment model required here.

### Signals Breakdown
- `HIGH_BUDGET` — $50-80/hr × 30+ hrs/week × 1-3 months = $6,000-$28,800+ total project value. Expert level.
- `DETAILED_SPEC` — Job post lists 15+ already-built features plus 10 specific backend requirements. 500+ words, very specific.
- `TECH_SPECIFIC` — Explicitly names Supabase/Postgres/Prisma, NextAuth/Clerk, Stripe, Vercel/AWS, WebSocket/SSE, WhatsApp Business API, GDPR. Stack matches Humam's exactly except Supabase (adjacent to existing REST/API experience).
- `CROWDED` — "20-50 proposals" listed. Demo becomes critical differentiator.
- `REDESIGN` — Explicitly: "take it from prototype to production." Not greenfield, not MVP — extending an existing codebase.

The cover letter and screening answers must address this specifically — not just mention tool names, but describe HOW the workflow changes (see screeningAnswer2 above).
